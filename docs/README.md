# Quadro Designer — Project Documentation

## Documents

| File | Purpose | Type |
|---|---|---|
| `../00_Idea.md` | Original idea, requirements, and project goals | Your input |
| `02_Roadmap.md` | Phased plan with success criteria | Living document |
| `03_Tech_Stack.md` | Technology choices and rationale | Living document |
| `04_Competitive_Analysis.md` | Existing tools, evaluation checklist | Living document |
| `05_Business_Plan.md` | Monetization, costs, risks | Living document |
| `06_Decision_Log.md` | Key decisions with context | Append-only |

## Conventions

- **Living documents** are always overwritten to reflect the current state. No versioned copies.
- **The Decision Log** is append-only. It captures the reasoning behind changes, providing traceability.
- **Code** is versioned via Git (once the project repository is set up).
